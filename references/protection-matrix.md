# 保护等级矩阵

根据环境检测结果确定实际可用的保护等级。

## 判定矩阵

| sbx | Docker Daemon | Git | 保护等级 | 建议方案 |
|---|---|---|---|---|
| ✅ | ✅ | ✅ | 🟢 完整 | 沙箱隔离 + 快照 + 分支（最优） |
| ✅ | ❌ | ✅ | 🟢 沙箱 | sbx --clone 隔离 + git 分支 |
| ✅ | ✅ | ❌ | 🟡 沙箱+快照 | sbx 隔离 + Docker 快照 |
| ✅ | ❌ | ❌ | 🟡 沙箱 | sbx 隔离（建议 git init） |
| ❌ | ✅ | ✅ | 🟢 标准 | Docker 快照 + git 分支 |
| ❌ | ✅ | ❌ | 🟡 快照 | 仅 Docker 快照 |
| ❌ | ❌ | ✅ | 🟡 降级 | 仅 git 分支 |
| ❌ | ❌ | ❌ | 🔴 无 | 强烈建议中止 |

## 各等级保护能力

### 🟢 完整保护（所有能力可用）
- **sbx 沙箱**：`--clone` 模式，宿主机只读
- **Docker 快照**：文件级备份
- **Git 分支**：代码版本保护

### 🟢 标准保护（Docker + Git）
- **Docker 快照**：文件级备份
- **Git 分支**：代码版本保护

### 🟡 降级保护（仅 Git）
- **Git 分支**：代码版本保护
- **无文件级快照**：回滚依赖 git

### 🔴 无保护（全部不可用）
- **仅警告**：强烈建议不要继续
- **建议先安装 git 或 Docker**

## 快照排除清单

创建 Docker 快照时排除以下目录（一致的排除列表）：

```
node_modules, .git, target, __pycache__, .claude, .venv, venv, dist, build, .next, out, *.log
```

## 检测优先级

```
sbx 可用？──是──→ 🟢 沙箱路径
    │
    否
    │
    ↓
Docker 运行中？──是──→ 🐳 Docker 路径
    │
    否
    │
    ↓
Git 仓库？──是──→ 🌿 仅分支路径
    │
    否
    │
    ↓
🔴 无保护
```
